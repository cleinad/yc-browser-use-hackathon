<!-- Key concerns for Vercel + bu-agent coexistence -->

## vercel.json
```json
{
  "framework": "nextjs",
  "installCommand": "npm install",
  "buildCommand": "npm run build",
  "outputDirectory": ".next"
}
```
This is a minimal explicit config. Do NOT add "rootDirectory" here — that is set in the Vercel dashboard during project import, not in vercel.json.

## .vercelignore
```
bu-agent/
docs/
MEGAPLAN.md
SPONSORS.MD
*.py
*.sh
requirements*.txt
```
This prevents Vercel from including Python backend files in the deployment bundle, keeping builds fast.

## Key pitfall: Root Directory vs. bu-agent/
During Vercel project import, the "Root Directory" field MUST be set to `/` (the repo root). If left as auto-detected or set to `bu-agent/`, Vercel will try to build the Python backend. The user must manually verify this during the checkpoint step.

  <how-to-verify>
    **Step A — Import repo to Vercel (if not already done):**
    1. Go to https://vercel.com/new
    2. Click "Import Git Repository"
    3. Select the yc-browser-use-hackathon repo
    4. CRITICAL: Set "Root Directory" to `/` (the repo root — where package.json lives)
    5. Verify "Framework Preset" is detected as "Next.js"
    6. Click "Deploy" (this first deploy will likely fail due to missing env var — that's OK)

    **Step B — Add NEXT_PUBLIC_CONVEX_URL environment variable:**
    1. Go to your Vercel project → Settings → Environment Variables
    2. Add: `NEXT_PUBLIC_CONVEX_URL` = the value from your local `.env.local` file (the `https://...convex.cloud` URL)
    3. Set it for: Production, Preview, and Development environments
    4. Click Save

    **Step C — Trigger a fresh deployment:**
    Either:
    - Go to Vercel project → Deployments → click the latest deployment → "Redeploy"
    - OR push a small commit to main: `git commit --allow-empty -m "chore: trigger Vercel deploy" && git push origin main`

    **Step D — Verify the deployment:**
    1. Watch the build logs in Vercel — should see "✓ Compiled" and no errors
    2. Open the production URL (shown in Vercel after deployment)
    3. Confirm:
       - Dark background (not white)
       - Text: "Connected. Houses in DB: 0" (NOT "Connecting to Convex...")
       - A styled indigo "Primary Button" is visible
    4. Confirm auto-deploy is set: Project Settings → Git → Production Branch = "main"

    **If you see "Connecting to Convex..." on the production URL:**
    - The NEXT_PUBLIC_CONVEX_URL env var wasn't picked up
    - Redeploy after confirming the env var is saved in Vercel dashboard

    **If build fails with Python errors:**
    - Vercel is building bu-agent/ instead of the Next.js app
    - Go to Project Settings → General → Root Directory — change it to `/`
    - Trigger a redeploy
  </how-to-verify>
